# 틀니케어(DentureCare) — 실제 앱 만들기 가이드

기존 프로토타입(`DentureCare.html`)을 **실제 폰에서 쓰는 앱**으로 바꾼 결과물입니다.

## 무엇이 바뀌었나 (핵심)

기존 앱은 데이터를 **Claude 전용 저장소(`window.storage`)** 에 저장하도록 돼 있었습니다. 이 저장소는 Claude 화면 안에서만 존재하기 때문에, 실제 폰이나 일반 브라우저에서 앱을 켜면 **입력한 환자 정보·루틴 기록이 껐다 켜는 순간 모두 사라지는** 상태였습니다.

이번에 한 일:

| 항목 | 내용 |
|---|---|
| **영구 저장** | `window.storage`를 기기 내장 저장소(`localStorage`)로 동작하도록 교체. 이제 앱을 껐다 켜도 데이터가 유지됩니다. (기존 5,000줄 코드는 그대로 재사용) |
| **설치형 앱(PWA)** | 홈 화면에 아이콘으로 설치되고, 전체화면·오프라인으로 실행되도록 매니페스트·서비스워커·앱 아이콘 추가 |
| **네이티브 프로젝트** | App Store / Play Store 출시까지 갈 수 있는 Capacitor 프로젝트 구성 |

> **검증 완료**: 저장→재시작→복원→삭제 동작을 테스트로 확인했고, 앱이 인터넷 없이 단독 실행되는 것(React·Babel 전부 내장)도 확인했습니다.

## 폴더 구조

```
Denture Care program/
├─ app/                       ← 실제 앱 본체 (PWA·네이티브 공용)
│  ├─ index.html              ← 저장소 교체 + PWA 적용된 앱
│  ├─ manifest.webmanifest    ← 앱 이름·아이콘·화면 설정
│  ├─ sw.js                   ← 오프라인 캐싱
│  └─ icons/                  ← 앱 아이콘 (192/512/마스커블/애플터치)
├─ capacitor.config.json      ← 네이티브 앱 설정
├─ package.json               ← 네이티브 빌드 의존성
└─ README_설치방법.md          ← 이 문서
```

# 방법 A. 오늘 바로 폰에서 써보기 (PWA, 추천)

가장 빠르고 비용 0원. 5분이면 폰 홈 화면에 앱이 깔립니다.

### 1단계 — 인터넷 주소(HTTPS)로 올리기
설치형 앱과 오프라인 기능은 보안주소(https)에서만 켜집니다. 무료 호스팅에 `app` 폴더만 올리면 됩니다.

**가장 쉬운 방법 — Netlify Drop (회원가입 없이):**
1. 브라우저에서 `https://app.netlify.com/drop` 접속
2. `app` **폴더를** 화면에 드래그&드롭
3. 잠시 후 `https://○○○.netlify.app` 주소가 생성됨 (이게 앱 주소)

> GitHub Pages, Vercel 등 다른 무료 호스팅도 동일하게 가능합니다.

### 2단계 — 폰에서 열고 홈 화면에 추가
생성된 주소를 폰 브라우저로 엽니다.

- **아이폰(Safari)**: 하단 공유 버튼 → **"홈 화면에 추가"**
- **안드로이드(Chrome)**: 우측 상단 ⋮ → **"앱 설치"** 또는 **"홈 화면에 추가"**

이제 홈 화면 아이콘으로 실행되며, 전체화면·오프라인으로 작동하고 데이터가 폰에 저장됩니다.

### (선택) 내 맥에서 먼저 미리보기
```bash
cd "app"
python3 -m http.server 8000
```
같은 와이파이의 폰에서 `http://<맥의 IP>:8000` 으로 접속해 동작 확인 가능.
(단, 홈 화면 설치·오프라인 기능은 위 HTTPS 주소에서만 완전히 켜집니다.)

# 방법 B. 진짜 네이티브 앱으로 빌드 (App Store / Play Store)

맥을 쓰시니 iOS·Android 둘 다 가능합니다. 같은 `app` 폴더를 네이티브 껍데기로 감싸는 방식이라 코드를 다시 짤 필요가 없습니다.

### 사전 준비
- **공통**: [Node.js](https://nodejs.org) 설치
- **iOS**: 맥 App Store에서 **Xcode** 설치 (무료 Apple ID로 본인 아이폰에 7일짜리 테스트 설치 가능 / 정식 출시는 연 $99 개발자 등록)
- **Android**: [Android Studio](https://developer.android.com/studio) 설치

### 빌드 명령 (프로젝트 폴더에서)
```bash
# 1) 의존성 설치 (최초 1회)
npm install

# 2) 플랫폼 추가 (원하는 것만)
npx cap add ios
npx cap add android

# 3) 웹 자산 동기화 (app 폴더 내용을 네이티브로 복사)
npx cap sync

# 4) 개발 도구 열기
npx cap open ios        # Xcode가 열림 → 아이폰 연결 → ▶ 실행
npx cap open android    # Android Studio가 열림 → 폰 연결 → ▶ 실행
```

앱을 수정할 때마다 `app/index.html`을 고친 뒤 `npx cap sync`만 다시 실행하면 됩니다.

### 앞으로의 과제 (네이티브 한정)
- **알림(가장 중요)**: 현재 앱의 알림은 웹 표준 `Notification`을 사용합니다. 폰이 잠겨있거나 앱이 꺼진 상태에서도 정해진 시각에 울리는 **진짜 예약 알림**은 네이티브 기능이 필요합니다. 이를 위해 `@capacitor/local-notifications` 플러그인을 미리 `package.json`에 넣어뒀고, 다음 단계로 루틴 시간(기상/식사/취침)·리콜(D-14, D-7 …)을 이 플러그인으로 예약하도록 연결하면 됩니다.
- **아이콘·스플래시**: `npx cap add` 후 `@capacitor/assets`로 `app/icons`의 아이콘을 네이티브 아이콘으로 자동 생성할 수 있습니다.

# 데이터 저장 정책 (현재)

- 모든 데이터는 **기기 안에만** 저장됩니다(`localStorage`). 서버·클라우드로 전송되지 않습니다.
- 따라서 보호자·치과와의 **실시간 연동은 아직 동작하지 않습니다**(화면 흐름은 구현돼 있으나 데이터는 기기 단위).
- 추후 실시간 연동이 필요하면 Firebase 등 클라우드 백엔드를 붙이는 단계로 확장하면 됩니다. 이때 환자 건강정보를 다루므로 동의·보안(암호화, 접근권한) 설계가 함께 필요합니다.
